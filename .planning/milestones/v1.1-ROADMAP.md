## Overview

v1.1 聚焦修复多实例 `skill-server / gateway` 场景下，消息被非真实连接实例消费，导致 miniapp 看不到 opencode 实时返回的问题。

## Phases

### Phase 6: 修复 skill server 与 gateway 多实例场景下消息被非连接实例消费，导致用户收不到 opencode 返回消息

**Goal**: 修复多实例下的消息消费归属错误，确保消息只被真实持有客户端流连接的 `skill-server` 实例消费  
**Depends on**: Phase 5  
**Plans**: 3 plans

Plans:

- [x] 06-01: 为 gateway / skill-server 链路补齐并校验可信 `userId` 上下文
- [x] 06-02: 将 `skill-server` 实时广播切换为 `user-stream:{userId}` 并建立首订阅 / 末退订生命周期
- [x] 06-03: 清理旧 `session:{sessionId}` 实时链路并补上多实例错投回归验证

**Details:**

- `GatewayMessage` 顶层新增 `userId`
- `gateway` 基于 `ak -> userId` 做 invoke 校验与回流补全
- `pc-agent` 不再携带 `userId`
- `skill-server` 实时广播从 `session:{sessionId}` 切换到 `user-stream:{userId}`
- `SkillStreamHandler` 负责用户级订阅生命周期
- 真实 UAT 中发现并修复了 WebSocket 双重清理导致的误退订问题

## Milestone Summary

**Key Decisions:**

- 继续复用 `GatewayMessage`，通过补充 `userId` 完成链路对齐
- 实时广播维度从 `sessionId` 调整为 `userId`
- `pc-agent` 不感知 `userId`
- 实时消费资格由用户流连接订阅关系决定，而不是由 REST 命中实例决定

**Issues Resolved:**

- 修复多实例下返回消息被错误实例消费的问题
- 修复 miniapp “刷新后才看到返回”的实时订阅误退订问题

**Issues Deferred:**

- `documents/gateway-skill-routing/README.md` 仍主要描述 v1.0 基线，后续可继续整合 v1.1 内容

**Technical Debt Incurred:**

- A/B/C 多实例联调仍主要依赖人工步骤，后续可继续脚本化
