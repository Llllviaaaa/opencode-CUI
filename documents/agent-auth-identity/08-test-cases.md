# 测试用例文档

> 版本：1.1  
> 日期：2026-03-09

## 一、后端单元测试

### 1.1 子协议认证

| 用例 ID    | 场景               | 操作                                                      | 期望结果                                     |
| ---------- | ------------------ | --------------------------------------------------------- | -------------------------------------------- |
| UT-AUTH-01 | 子协议认证成功     | 发送有效 `Sec-WebSocket-Protocol: auth.{valid-base64url}` | 握手成功，回应子协议 `auth.{same-base64url}` |
| UT-AUTH-02 | 子协议缺失         | 不发送 `Sec-WebSocket-Protocol`                           | 握手拒绝                                     |
| UT-AUTH-03 | 子协议格式错误     | 发送 `Sec-WebSocket-Protocol: invalid`                    | 握手拒绝                                     |
| UT-AUTH-04 | Base64URL 解码失败 | 发送 `auth.!!!invalid!!!`                                 | 握手拒绝                                     |
| UT-AUTH-05 | JSON 格式错误      | Base64 编码的非 JSON 字符串                               | 握手拒绝                                     |
| UT-AUTH-06 | 签名校验失败       | 有效格式但 sign 错误                                      | 握手拒绝                                     |
| UT-AUTH-07 | AK 不存在          | 未注册的 AK                                               | 握手拒绝                                     |
| UT-AUTH-08 | 时间戳过期         | ts 超出窗口期                                             | 握手拒绝                                     |

### 1.2 注册三步校验

| 用例 ID   | 场景                 | 操作                        | 期望结果                                                       |
| --------- | -------------------- | --------------------------- | -------------------------------------------------------------- |
| UT-REG-01 | 正常注册             | 发送有效 register 消息      | 收到 `register_ok`，Agent 状态 ONLINE                          |
| UT-REG-02 | 设备绑定校验失败     | 绑定服务返回 `valid: false` | 收到 `register_rejected("device_binding_failed")`，close(4403) |
| UT-REG-03 | 设备绑定服务超时     | 绑定服务 >3s 不响应         | **降级放行**，正常注册                                         |
| UT-REG-04 | 设备绑定服务不可用   | 绑定服务 URL 不可达         | **降级放行**，正常注册                                         |
| UT-REG-05 | 设备绑定功能未启用   | `enabled=false`             | 直接通过，正常注册                                             |
| UT-REG-06 | 重复连接拒绝         | 同 AK 已有活跃 session      | 收到 `register_rejected("duplicate_connection")`，close(4409)  |
| UT-REG-07 | 旧连接已断开后新连接 | 旧连接已 close 后再连       | 正常注册，不被拒绝                                             |

### 1.3 注册超时

| 用例 ID       | 场景     | 操作                          | 期望结果                |
| ------------- | -------- | ----------------------------- | ----------------------- |
| UT-TIMEOUT-01 | 注册超时 | 握手成功后 >10s 不发 register | 连接被关闭，close(4408) |
| UT-TIMEOUT-02 | 及时注册 | 握手后 <10s 发送 register     | 正常注册，不被超时      |

### 1.4 身份持久化

| 用例 ID  | 场景                 | 操作                            | 期望结果                            |
| -------- | -------------------- | ------------------------------- | ----------------------------------- |
| UT-ID-01 | 首次注册             | 新 AK + toolType                | INSERT 新记录                       |
| UT-ID-02 | 再次注册（身份复用） | 同 AK + toolType，旧记录已存在  | UPDATE 已有记录，agentId 不变       |
| UT-ID-03 | 不同 toolType        | 同 AK，不同 toolType            | INSERT 新记录（不同 toolType 独立） |
| UT-ID-04 | 设备信息更新         | 同 AK+toolType，deviceName 变化 | 更新 deviceName，agentId 不变       |

### 1.5 GatewayMessage 序列化

| 用例 ID   | 场景                      | 操作                                        | 期望结果                                      |
| --------- | ------------------------- | ------------------------------------------- | --------------------------------------------- |
| UT-MSG-01 | register 工厂方法         | `GatewayMessage.register(...)`              | 包含 macAddress 字段                          |
| UT-MSG-02 | registerOk 工厂方法       | `GatewayMessage.registerOk()`               | `type="register_ok"`                          |
| UT-MSG-03 | registerRejected 工厂方法 | `GatewayMessage.registerRejected("reason")` | `type="register_rejected"`, `reason="reason"` |

## 三、集成测试

### 3.1 正常连接闭环

| 用例 ID | 场景         | 步骤                                                | 期望结果                                          |
| ------- | ------------ | --------------------------------------------------- | ------------------------------------------------- |
| IT-01   | 完整连接流程 | PC Agent 启动 → 子协议握手 → register → register_ok | Agent 状态 ONLINE，Skill Server 收到 agent_online |
| IT-02   | 断开重连     | 关闭连接 → 自动重连 → 重新 register                 | 同一 agentId，状态恢复 ONLINE                     |
| IT-03   | 心跳保活     | 连接后等待 >30s                                     | 心跳正常，连接不断开                              |

### 3.2 安全场景

| 用例 ID   | 场景           | 步骤                          | 期望结果                  |
| --------- | -------------- | ----------------------------- | ------------------------- |
| IT-SEC-01 | URL 无敏感参数 | 检查 Gateway/Nginx 日志       | URL 中没有 ak/sign 等参数 |
| IT-SEC-02 | 旧版客户端兼容 | 旧版 PC Agent（URL 认证）连接 | 握手被拒绝（无子协议）    |

### 3.3 重复连接与拒绝

| 用例 ID   | 场景             | 步骤                           | 期望结果                    |
| --------- | ---------------- | ------------------------------ | --------------------------- |
| IT-DUP-01 | 同 AK 重复连接   | 两个 PC Agent 同时连接同一 AK  | 第一个正常，第二个收到 4409 |
| IT-DUP-02 | 旧连接关闭后再连 | 关闭第一个 → 等待 → 启动第二个 | 第二个正常连接              |

### 3.4 身份持久化

| 用例 ID  | 场景                | 步骤                              | 期望结果                      |
| -------- | ------------------- | --------------------------------- | ----------------------------- |
| IT-ID-01 | 重启后 agentId 稳定 | 连接 → 记录 agentId → 断开 → 重连 | agentId 相同                  |
| IT-ID-02 | 历史会话可复用      | 重启后查询历史 skill_session      | 关联到同一 agentId 的会话存在 |
